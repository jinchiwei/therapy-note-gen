# Therapy Note Generator (prototype)

Turns a session transcript into a progress note in one clinician's house style.
The transcript is **de-identified locally** (Microsoft Presidio) before any text
is sent to the cloud model, then the note is generated by Claude.

```
transcript ──▶ Presidio de-id ──▶ [PERSON_1] placeholders ──▶ Claude ──▶ note ──▶ re-id
              (on this machine)                              (cloud API)
```

## Honest compliance note (read this)

- Automated de-identification of free-text therapy transcripts is **leaky**. It
  reliably strips names, dates, and phone numbers; it does **not** catch every
  narrative quasi-identifier ("my home country", a specific life event). It
  **reduces** risk but is not, on its own, HIPAA Safe-Harbor de-identification.
- This is a real improvement over pasting raw PHI into the consumer ChatGPT app.
- To be properly compliant, sign a **BAA** with your model provider (Anthropic
  and OpenAI both offer them for business/API accounts). De-id is then a second
  layer, not the only one.
- The app runs on `127.0.0.1` only, keeps the transcript and mapping in memory
  for a single request, and writes nothing to disk.

## Setup

Python 3.9+.

```bash
cd note-gen
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python -m spacy download en_core_web_lg

cp .env.example .env        # then put your real ANTHROPIC_API_KEY in .env
```

> On Jin's Mac you can install into the `deepdream` conda env instead of a venv.

## Run

```bash
# load the API key (or use a dotenv loader / set it in your shell)
export $(grep -v '^#' .env | xargs)
python app.py
```

Open <http://127.0.0.1:5005>, paste a transcript, click **Generate note**, copy
the result. The "What was redacted" panel shows exactly which spans were
replaced before sending — review it to confirm nothing identifying slipped past.

## Quick CLI test (no web UI)

```bash
python -c "from deid import deidentify; t=open('sample_transcript.txt').read(); d,m=deidentify(t); print(d); print(m)"
```

## Files

| File | Purpose |
|------|---------|
| `deid.py` | Presidio de-id / re-id with reversible in-memory mapping |
| `notegen.py` | Claude call + the house-style system prompt (defaults to Opus 4.8) |
| `examples.py` | **Synthetic** few-shot exemplar notes (no real patient material) |
| `app.py` | Local Flask web UI |
| `sample_transcript.txt` | Synthetic transcript for testing (no real PHI) |
| `sample_output.md` | Illustrative generated note from that transcript |

## Using your own (real) examples — locally only

The committed `examples.py` is fully synthetic so the repo can be public. For
maximum fidelity to your real house style, create an **`examples_local.py`** that
defines an `EXEMPLARS` list of your own de-identified notes — `notegen.py` uses it
automatically when present. It is gitignored.

> **Never commit real patient material, even de-identified.** Keep it in
> `examples_local.py` and let `.gitignore` do its job.

## Tuning

- **Over-redaction** (a real word turned into a placeholder): raise the
  `threshold` in `deid.py` (default `0.4`) or drop an entity from
  `DEFAULT_ENTITIES`.
- **Under-redaction** (something slipped through): lower the threshold, or add a
  custom Presidio recognizer.
- **Note style drift**: edit `SYSTEM_PROMPT` in `notegen.py` or add more
  exemplars to `examples.py`.
